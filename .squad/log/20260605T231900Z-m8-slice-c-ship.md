## Scope

**Wrapped-Cursor Q2 Form:** FactStore search API returns `{ results: RecallResult[], nextCursor: string | null }` enabling offset-based pagination without exposing raw OFFSET/LIMIT semantics.

**FTS5 BM25 Search:** SqliteFactStore integrates SQLite's built-in FTS5 (Full-Text Search 5) with BM25 relevance scoring. Query: `SELECT f.*, rank() OVER (ORDER BY fts.rank) AS fts_rank FROM facts_fts fts JOIN facts f ON fts.rowid = f.id ORDER BY fts.rank, f.id ASC LIMIT ?`. Tied results break on `f.id ASC` (insertion order, autoincrement INTEGER PRIMARY KEY).

**Session-Scoped FactStore:** Ledger snapshot at fact-insertion time; facts are immutable within a session. No cross-session queries (session boundary is write-scoped).

**minTrust Validation:** Constructor rejects `minTrust < 0 || minTrust > 1`; throws `TypeError` (API contract violation).

**Cursor Validation (v1 Offset):** API validates `limit > 0` and `limit !== Infinity`; rejects `limit=0` and `limit=-1` with `TypeError`. No scope fingerprinting in v1 (deferred to Slice D); offset cursor is not bound to query params/session/minTrust/limit — cross-parameter reuse is undefined behavior (documented in comments).

**Tie-Break Alignment:** Both SqliteFactStore and InMemory reference impl sort tied results by insertion order. SqliteFactStore uses `f.id ASC` (autoincrement PK); InMemory uses explicit `insertionOrder` counter (incremented on each `seed()` call).

## Review-Cycle Findings

**Local Review:** 2 persona-review rounds + Code Panel  
- **F1 (Relevance Semantics):** `relevance` is normalized `-bm25` per-page, NOT composite score. `compositeScore` consumer weights relevance, trust, importance, recency orthogonally (each has coefficient). Baking trust into relevance would double-count.
- **F2 (FTS5 Catch):** FTS5 error message narrowing required both error code AND message regex pattern. Code-only (`code === 'SQLITE_ERROR'`) would over-swallow missing-table errors (`"no such table: facts_fts"`). Shipped pattern: `/fts5|unterminated|syntax error|malformed MATCH/i`.
- **F3 (Tie-Breaker):** `f.id ASC` is cheap (BTree INTEGER PK cost ≈ 0); InMemory mirrors with `insertionOrder` counter, not `localeCompare` (would produce wrong order for non-alphabetical insertion).
- **F4 (Limit Validation):** `limit=0` produces infinite pagination loop; `limit=-1` is SQLite implementation quirk (unlimited). Both are API contract violations, signal `TypeError`.
- **C2-A/B/C/D/E (Cloud Review):** Cursor validation, ledger reconciliation, contract tests distinguishing implementations.
