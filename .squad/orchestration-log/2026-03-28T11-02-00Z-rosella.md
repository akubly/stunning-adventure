### Plugin Categories

1. **Skill Bundles** — SKILL.md format, cross-platform (CLI + VS Code + GitHub.com)
2. **Agent Starters** — AGENTS.md format, vendor-neutral, orchestration templates
3. **Tool Integrations** — MCP servers, registry-backed discovery
4. **Hook Extensions** — User-scoped workflows, pre/post-action hooks
5. **Marketplace Add-ons** — awesome-copilot submissions, community templates

## Key Design Decisions

- **Keep full compiler** — Plugin system compiles to executable agents/skills; no runtime interpretation limits
- **BYO plugin authoring** — Developers can author plugins in TypeScript, Python, Go; we handle compilation + distribution
- **Adapter pattern** — Interop layer between plugins, agents, squad, plan mode, SA (Structured Autonomy)
- **No custom marketplace** — Integrate with awesome-copilot, MCP Registry; don't rebuild

## Context Shared

- User directive: Personalization as first-class, BYO interop with other agentic systems
- Team direction: Plugin-first, marketplace integration
- Marketplace landscape: awesome-copilot dominance, SKILL.md/AGENTS.md as emerging standards

## Technical Insights

- Keeping full compiler prevents vendor lock-in; plugins stay portable
- Adapter pattern is lighter than platform abstraction; maps plugin outputs → system inputs
- awesome-copilot ecosystem is mature; no need to reinvent

## Tensions Explored

- **Compiler complexity vs. flexibility:** Keeping full compiler enables BYO; worth the complexity
- **Plugin format standardization:** Choose SKILL.md for skills, AGENTS.md for agents; simple, proven
- **Interop depth:** Adapters handle squad, plan mode, SA; broader integrations can come later

## Team Sentiment

Strong support for plugin-first approach. Team appreciated that BYO doesn't mean "host weird formats" — standardize on SKILL.md and AGENTS.md.
