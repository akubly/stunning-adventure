## Local Review Cycle Findings

**F1 — Relevance Semantics:** Clarified that `relevance` is normalized `-bm25` per-page only, not composite score. Composite weighting happens downstream in consumer, preserving orthogonality of signals (relevance, trust, importance, recency).

**F2 — FTS5 Error Narrowing:** Initial logic (`code === 'SQLITE_ERROR'`) was too broad. Real SQLite testing showed missing-table errors (`"no such table: facts_fts"`) are also `SQLITE_ERROR`. Shipped fix: both code AND message regex pattern `/fts5|unterminated|syntax error|malformed MATCH/i`.

**F3 — Tie-Breaker:** Verified `f.id ASC` cost ≈ 0 on INTEGER PK (BTree). InMemory reference impl mirrors with explicit `insertionOrder` counter (not `localeCompare`, which would produce wrong order for non-alphabetical insertion).

**F4 — Limit Validation:** `limit=0` and `limit=-1` are API contract violations. `limit=0` produces infinite loop (LIMIT 0 always returns 0, `nextCursor` defined, repeat forever). Shipped fix: `TypeError` signal.

**C2-A through C2-E (Cloud Review):** Ledger reconciliation, cursor contract alignment, contract tests distinguishing implementations (FS-7: non-alphabetical seed data `tie-c/a/b` + Set uniqueness assertion).
