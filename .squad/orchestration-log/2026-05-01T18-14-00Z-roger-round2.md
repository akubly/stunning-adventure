**Outcome:** Storage Pattern Validated

### 3. Caching at the Data Layer: Upstream vs. Downstream Filtering

**Outcome:** Tension Documented, Resolution Deferred to Aaron

**My Position (Upstream Filtering):**
- Filter data at source (tool invocation layer, event bridge)
- Fewer artifacts to store = simpler queries, lower storage costs, faster retrieval
- Reduced noise in ancestry chains
- Risk: Discarding potentially relevant context; harder to backfill if filtering rules change

**Alexander's Position (Downstream Filtering):**
- Capture everything into cache layers
- Maximize detail for future analysis (pattern extraction, genetic programming training)
- Filter on read (queries, skill export, dashboard views)
- Supports Aaron's prior guidance: "Why would we not want as much detail as possible?"
- Risk: Unbounded storage growth; requires retention policies and compression

**Aaron's Directive:** Maximum detail preferred. Tradeoffs to be articulated explicitly before any future pruning decisions.

**Resolution Path:**
1. Phase 4.5: Implement downstream (capture everything). Monitor storage metrics.
2. Phase 5: If storage becomes bottleneck, implement time-based retention (archive at 1yr, compress).
3. Future: Revisit filtering rules based on empirical data (what do we actually use?).

### 4. Cache Layer Hierarchy Integration

**Outcome:** Caching Strategy with Data Layer Implications

L1-L4 hierarchy applied at data layer:
- **L1 (In-Memory):** Tool result memoization. Prevents re-fetching same artifact within turn.
- **L2 (Session Store):** Event bridge caches provenance classifications. Semantic fingerprinting enables cache hits across similar events.
- **L3 (Short-TTL):** Cached query results (ancestry chains, knowledge graph traversals). ~1 hour reuse window.
- **L4 (Long-TTL):** Archived artifacts for pattern extraction. ~30 day window before compression/deletion.

**Storage Projection:** Phase 4.5 canary (10 users, 100 sessions/user) = ~50-100 MB (uncompressed). Phase 5 production (1000 users) = ~5-10 GB without retention policies. Phase 5 to implement archival + compression.
